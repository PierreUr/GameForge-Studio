Willkommen bei GameForge Studio!

GameForge Studio ist eine webbasierte Umgebung, die es Ihnen ermöglicht, auf einfache Weise 2D-Spiele und Spielmodi zu erstellen. Als **Template Admin** ist es Ihre Hauptaufgabe, aus *vorhandenen* Bausteinen (Components, Logic Nodes, Assets) neue Spielvarianten, Level oder komplette Spielmodi ("Custom Games") zu konfigurieren.

Dieses Handbuch führt Sie durch die Werkzeuge und Konzepte, die Sie benötigen, um Ihre eigenen Kreationen in GameForge Studio zu verwirklichen, ohne selbst Code schreiben zu müssen.

Hinweis für Super Admins: Ein separater Abschnitt am Ende dieses Dokuments (Für Super Admins) beschreibt, wie neue Bausteine (Components, Nodes, Assets) entwickelt und in das System integriert werden können.

Überblick über die Benutzeroberfläche

Die Oberfläche ist in vier Hauptbereiche aufgeteilt:

Platzhalter: Bild von GameForge Studio Benutzeroberfläche
  1. **Toolbar (Oben):** Globale Aktionen (Speichern, Laden, Undo/Redo etc.).
  2. **Left Sidebar (Links):** Werkzeugkasten mit Bausteinen (Library, Logic Graph etc.). Anpassbare Breite.
  3. **Canvas / Main Content Area (Mitte):** Visueller Editor für Ihre Spielszene.
  4. **Right Sidebar (Rechts):** Detailansicht und Einstellungen für ausgewählte Elemente (Inspector). Anpassbare Breite.

Toolbar

  • Undo/Redo: Aktionen rückgängig machen oder wiederholen.
  • Save Project: Speichert das Projekt als .json-Datei.
  • Load Project: Lädt ein Projekt aus einer .json-Datei (überschreibt aktuellen Stand).
  • Export HTML: Erzeugt eine spielbare HTML-Datei (Coming Soon).
  • Toggle Grid: Schaltet das Hilfsraster im Canvas an/aus.
  • Toggle Colliders: Zeigt/Versteckt Kollisionsboxen zur Fehlersuche.
  • Live Preview: (Coming Soon) Startet eine Spielvorschau.

Left Sidebar

  • Library Tab: Enthält ComponentCards - visuelle Repräsentationen von Templates (vorgefertigte Entity-Typen) und einzelnen Components. Ziehen Sie diese auf das Canvas.
  • Layers Tab: (Coming Soon) Zukünftige Verwaltung von Zeichenebenen.
  • Logic Graph Tab: Visueller Editor zur Programmierung der Spiellogik mit Knoten.
  • Assets Tab: (Coming Soon) Zukünftige Verwaltung von Bildern, Sounds etc.

Canvas / Main Content Area

  • Zeigt die Spielszene.
  • **Navigation:** Mittlere Maustaste gedrückt halten zum Verschieben (Pan), Mausrad zum Zoomen.
  • **Interaktion:** Linksklick zum Auswählen einer Entity, Entf/Rücktaste zum Löschen der Auswahl. Drag & Drop von Templates aus der Library zum Erstellen.

Right Sidebar

  • Inspector Tab: Zeigt die Components (Eigenschaften) der ausgewählten Entity an und erlaubt deren Bearbeitung. Änderungen sind sofort sichtbar und können rückgängig gemacht werden.

Entities & Components verstehen

GameForge Studio nutzt ein Baukastensystem:

  • Entities: Die Objekte in Ihrer Spielwelt (Spieler, Gegner, Münzen). Sie sind im Grunde nur IDs.
  • Components: Die Eigenschaften, die einer Entity Leben einhauchen. Eine Entity ist die Summe ihrer Components. Beispiele:
    • PositionComponent: Gibt der Entity einen Ort.
    • SpriteComponent: Gibt der Entity ein Aussehen.
    • HealthComponent: Ermöglicht es der Entity, Schaden zu nehmen.
    • ColliderComponent: Ermöglicht Kollisionen.

Als Template Admin kombinieren Sie vorhandene Components, um das gewünschte Verhalten und Aussehen für Ihre Entities zu erreichen.

Spielmodi & Templates erstellen (als Template Admin)

Ihre Hauptaufgabe als Template Admin ist es, mithilfe der vorhandenen Werkzeuge neue Spielmodi, Level oder wiederverwendbare Entity-Vorlagen ("Templates") zu erstellen.

1. Entities erstellen und konfigurieren

  • **Aus der Library ziehen:** Der einfachste Weg, Entities zu erstellen, ist das Ziehen von fertigen Templates (wie Player, Enemy) aus dem **Library Tab** auf das **Canvas**. Diese Templates bringen bereits eine sinnvolle Kombination von Components mit.
  • **Components anpassen:** Wählen Sie eine erstellte Entity im **Canvas** aus. Im **Inspector** (rechte Seitenleiste) sehen Sie nun alle Components dieser Entity. Ändern Sie hier die Werte nach Bedarf (z.B. die maxHealth im HealthComponent, die texture im SpriteComponent oder die waypoints im AIPatrolComponent).
  • **(Coming Soon): Components hinzufügen/entfernen:** Zukünftig wird es möglich sein, auch einzelne Components aus der Library zu einer bestehenden Entity hinzuzufügen oder davon zu entfernen, um hochgradig angepasste Entities zu erstellen.

2. Szene anordnen

Positionieren Sie Ihre erstellten Entities im **Canvas** durch Anpassen der PositionComponent im Inspector, um Ihr Level oder Ihre Spielszene zu gestalten. (Drag & Drop Positionierung im Canvas ist Coming Soon).

3. Logik definieren

Verwenden Sie den **Logic Graph**, um das Zusammenspiel der Entities und die Spielregeln festzulegen (siehe nächster Abschnitt).

4. Speichern

Speichern Sie Ihre Konfiguration regelmäßig über Save Project in der Toolbar. Diese gespeicherte .json-Datei repräsentiert Ihren erstellten Spielmodus oder Ihr Template.

Logic Graph nutzen (als Template Admin)

Der Logic Graph ermöglicht es Ihnen, auf Ereignisse im Spiel zu reagieren und Aktionen auszulösen, ohne Code zu schreiben. Sie verwenden dabei die von Super Admins bereitgestellten Knoten.

Coming Soon: Die volle Funktionalität des Logic Graphs, insbesondere das Verbinden von Knoten per Drag & Drop und die tatsächliche Ausführung der Logik, befindet sich noch in Entwicklung.

  • **Verfügbare Knoten finden:** Im **Logic Graph Tab** (linke Seitenleiste) sehen Sie eine Vorschau der verfügbaren Knoten (Events, Actions, Logik etc.).
  • **Konzept (Zukünftig):**
    1. Ziehen Sie benötigte Knoten (z.B. OnCollisionEvent, ModifyHealthAction) in den Graphen-Editor.
    2. Verbinden Sie die Ports der Knoten per Drag & Drop:
      • Flow Ports (Pfeile): Bestimmen die Reihenfolge der Ausführung.
      • Data Ports (Kreise): Übergeben Daten (z.B. Entity-IDs, Zahlenwerte).
    3. Konfigurieren Sie die Knoten im Inspector (falls Parameter vorhanden).

Assets nutzen (als Template Admin)

Coming Soon: Das Asset Management System befindet sich noch in Entwicklung. Die Funktionalität zum Hochladen, Verwalten und einfachen Auswählen von Assets ist noch nicht implementiert.

Assets sind die Medieninhalte Ihres Spiels (Bilder, Sounds).

  • **Konzept (Zukünftig):** Super Admins laden Assets über den **Assets Tab** hoch. Als Template Admin können Sie diese Assets dann über eine Auswahlbox oder per ID in Components wie SpriteComponent auswählen.
  • **Aktueller Stand:** Momentan müssen Asset-Pfade (z.B. player.png) direkt als Text in die Component-Felder eingetragen werden. Die Assets müssen manuell im Projektordner verfügbar sein.

Spielregeln & Berechnungen definieren

Coming Soon: Dedizierte Werkzeuge zur einfachen Konfiguration komplexer Spielregeln und Berechnungen (z.B. Schaden, Punkte, Sieg/Niederlage) sind in Planung.

Ein zentrales Ziel für Template Admins ist es, Spielmechaniken anzupassen, ohne Code zu schreiben. Zukünftig wird GameForge Studio dedizierte Oberflächen oder spezielle Components/Nodes bieten, um Folgendes zu konfigurieren:

  • Sieg- und Niederlagebedingungen

Projektverwaltung

Die Toolbar bietet die wesentlichen Funktionen zur Verwaltung Ihres Projekts:

  • **Speichern (Save Project):** Sichert den gesamten Zustand (Szene, Logik) in einer .json-Datei. Wichtig: Speichern Sie häufig!
  • **Laden (Load Project):** Öffnet ein zuvor gespeichertes Projekt. Ihr aktueller Stand wird überschrieben.
  • **Export (Export HTML):** (Coming Soon) Erzeugt eine eigenständige HTML-Version Ihres Projekts.
  • **Auto-Save:** Ihr Projekt wird alle 30 Sekunden automatisch im Browser zwischengespeichert, um Datenverlust zu minimieren. Dies ersetzt kein manuelles Speichern!

Responsive Design

GameForge Studio selbst passt sich an verschiedene Bildschirmgrößen an. Die exportierten Spiele (via Export HTML) sollen ebenfalls auf verschiedenen Geräten (Desktop, Tablet, Mobile) funktionieren. Die zugrundeliegende Engine ist dafür ausgelegt, die konkrete Umsetzung hängt aber vom zukünftigen Game Runner ab.

Coming Soon: Werkzeuge zur spezifischen Anpassung des Layouts und der Steuerung für verschiedene Geräte (Desktop, Tablet, Mobile) sind geplant.

Für Super Admins: System erweitern

Dieser Abschnitt richtet sich an Entwickler (Super Admins), die GameForge Studio um neue Bausteine erweitern möchten.

Coming Soon: Mechanismen zum dynamischen Laden von extern entwickelten Modulen (Plugins) sind geplant, um die Erweiterbarkeit zu verbessern.

Aktueller Stand: Die folgenden Anleitungen beschreiben, wie neue Elemente *direkt in den Kern-Code* von GameForge Studio integriert werden müssen.

Neue Components hinzufügen

  1. **TypeScript-Klasse erstellen:** In src/core/library/, von ComponentBase erben, Properties definieren, exportieren, TSDoc hinzufügen.
  2. **Manifest aktualisieren (src/core/assets/component-manifest.json):** Neuen Eintrag im components-Array hinzufügen (Name, Description, Properties mit Name, Typ, Beschreibung, DefaultValue). Der type bestimmt das UI-Control im Inspector.
  3. **World-Registrierung (index.tsx):** Neue Component importieren und mit world.registerComponent(MyNewComponent); registrieren.

Neue Logic Graph Nodes hinzufügen

  1. **TypeScript-Klasse erstellen:** In src/core/graph/nodes/, von NodeBase erben. Im Konstruktor Ports via this.addInput() / this.addOutput() definieren. Exportieren, TSDoc hinzufügen. (Execute-Logik ist Coming Soon).
  2. **Node Factory registrieren (src/core/graph/nodeFactory.ts):** Neue Node-Klasse importieren und in nodeRegistry-Map eintragen (nodeRegistry.set('MyNewNode', MyNewNode);).
  3. **(Optional) Visuelle Darstellung (src/core/ui/LogicGraphPanel.tsx):** Manuell Instanz hinzufügen für Vorschau (dynamische Library ist Coming Soon).

Neue Assets hinzufügen

Coming Soon: Das Asset Management System inklusive Upload, Verwaltung und Auswahl-UI ist in Entwicklung.

  1. **Konzept (Zukünftig):** Upload über "Assets Tab", Registrierung in Asset-DB/Manifest, Referenzierung per ID/Pfad in Components.
  2. **Aktueller Stand:** Assets manuell im Projekt ablegen, Pfade direkt in Component-Felder eintragen.

Allgemeine Regeln für Kern-Entwicklung

  • **ECS-Konformität:** Strikte Trennung von Daten (Components) und Logik (Systems).
  • **TypeScript:** Typisierung, Interfaces, TSDoc.
  • **Basisklassen:** ComponentBase, SystemBase, NodeBase nutzen.
  • **Metadaten:** component-manifest.json pflegen.
  • **Registrierung:** Neue Elemente in World, nodeFactory, SystemManager bekannt machen.

Tastaturkürzel

  • Strg+S / Cmd+S: Projekt speichern.
  • Strg+Z / Cmd+Z: Letzte Aktion rückgängig machen.
  • Strg+Y / Cmd+Shift+Z: Letzte rückgängig gemachte Aktion wiederholen.
  • Entf / Rücktaste: Löscht die aktuell ausgewählte Entity im Canvas.
